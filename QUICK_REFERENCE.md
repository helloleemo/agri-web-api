## 📊 表格速查表 (Table Quick Lookup)

### 1️⃣ 核心表格 (3 個)

```
┌─────────────────────────────────────────────────────────────┐
│ users (用戶表) - 9 個欄位                                    │
├─────────────────────────────────────────────────────────────┤
│ 主鑰匙: id (UUID)                                           │
│ 外鍵: role_code → roles, status_code → statuses            │
│ 唯一: email                                                  │
│ 索引: idx_users_email, idx_users_role_code, ...            │
│ 典型記錄: 100-10,000                                        │
│ 重要欄位:                                                    │
│   - email: 用戶郵件                                         │
│   - password_hash: 密碼雜湊                                 │
│   - role_code: 角色 (1=Admin, 2=Customer)                 │
│   - status_code: 狀態 (1=Active, 0=Inactive)              │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ products (產品表) - 11 個欄位                                │
├─────────────────────────────────────────────────────────────┤
│ 主鑰匙: id (UUID)                                           │
│ 外鍵: category_id → categories, status_code → statuses     │
│ 索引: idx_products_name, idx_products_category_id, ...     │
│ 約束: price >= 0, stock >= 0                              │
│ 典型記錄: 50-5,000                                         │
│ 重要欄位:                                                    │
│   - price: 基礎價格 (分)                                   │
│   - stock: 基礎庫存                                         │
│   - low_stock_threshold: 低庫存警告                        │
│   - category_id: 分類                                      │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ categories (分類表) - 4 個欄位                               │
├─────────────────────────────────────────────────────────────┤
│ 主鑰匙: id (UUID)                                           │
│ 唯一: name                                                   │
│ 典型記錄: 5-100                                            │
│ 重要欄位:                                                    │
│   - name: 分類名稱                                         │
│   - meta_data: JSONB 元資料                                │
└─────────────────────────────────────────────────────────────┘
```

### 3️⃣ 庫存表格 (2 個)

```
┌─────────────────────────────────────────────────────────────┐
│ inventory_balances (庫存結餘表) - 8 個欄位                   │
├─────────────────────────────────────────────────────────────┤
│ 主鑰匙: id (UUID)                                           │
│ 外鍵: product_id, unit_id                                 │
│ 複合唯一: (product_id, unit_id)                           │
│ 約束: actual_stock >= reserved_stock                       │
│ 典型記錄: 50-50,000 (每個 product-unit 一筆)              │
│ 重要欄位:                                                    │
│   - actual_stock: 實際庫存                                 │
│   - reserved_stock: 預留庫存                               │
│   - available = actual - reserved                          │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ inventory_ledger (庫存帳簿表) - 16 個欄位 ⭐ 最快增長      │
├─────────────────────────────────────────────────────────────┤
│ 主鑰匙: id (UUID)                                           │
│ 外鍵: product_id, unit_id, order_id, order_item_id, ...  │
│ 索引: idx_inventory_ledger_product_unit, ...              │
│ 典型記錄: 100,000-1,000,000+ (1-10 GB 以上)               │
│ 備註: 最重要的審計表，需要分區和歸檔                      │
│ 重要欄位:                                                    │
│   - action: 動作 (reserve, release, fulfill, return)     │
│   - delta_actual: 實際庫存變化                             │
│   - delta_reserved: 預留庫存變化                           │
│   - operator_user_id: 操作人員                             │
└─────────────────────────────────────────────────────────────┘
```

## 🔍 常用查詢 (Common Queries)

### 1. 查詢用戶的訂單
```sql
SELECT * FROM orders 
WHERE user_id = ? 
ORDER BY created_at DESC 
LIMIT 20;
```

### 2. 查詢產品各單位的庫存
```sql
SELECT u.name, pb.actual_stock, pb.reserved_stock,
       (pb.actual_stock - pb.reserved_stock) AS available
FROM product_units pu
JOIN units u ON pu.unit_id = u.id
LEFT JOIN inventory_balances pb 
  ON pu.product_id = pb.product_id AND pu.unit_id = pb.unit_id
WHERE pu.product_id = ?;
```

### 3. 查詢訂單完整資訊
```sql
SELECT o.*, oi.*, p.name, u.name
FROM orders o
LEFT JOIN order_items oi ON o.id = oi.order_id
LEFT JOIN products p ON oi.product_id = p.id
LEFT JOIN units u ON oi.unit_id = u.id
WHERE o.id = ?;
```

### 4. 查詢庫存變動歷史
```sql
SELECT * FROM inventory_ledger
WHERE product_id = ? AND unit_id = ?
ORDER BY created_at DESC
LIMIT 50;
```

### 5. 查詢用戶購物車
```sql
SELECT ci.*, p.name, p.price, u.name
FROM cart_items ci
JOIN products p ON ci.product_id = p.id
LEFT JOIN units u ON ci.unit_id = u.id
WHERE ci.cart_id = (
    SELECT id FROM carts WHERE user_id = ?
);
```

## 📈 效能提示 (Performance Tips)

### ✅ 做這些
- ✓ 使用索引進行查詢 (外鍵、status_code、created_at)
- ✓ 在 WHERE 子句中使用分頁 (LIMIT, OFFSET)
- ✓ 定期歸檔舊訂單和庫存記錄
- ✓ 使用複合索引進行多條件查詢
- ✓ 在事務中執行相關操作

### ❌ 不要做這些
- ✗ 全表掃描大表 (特別是 orders, inventory_ledger)
- ✗ 在 WHERE 子句中使用 LIKE '%...' (需要索引)
- ✗ 一次加載大量記錄到記憶體
- ✗ 在 JON 中使用 OR (改用 IN)
- ✗ 忽略數據完整性約束

## 🚨 重要提醒 (Important Notes)

### 金額儲存
- 所有金額以 **分** (¥0.01) 為單位
- 100 分 = ¥1.00
- **不使用浮點數** (避免精度問題)

### 時區
- 所有 TIMESTAMP 使用 **UTC+0 (Zulu Time)**
- 應用層需要轉換為本地時間

### 級聯刪除
⚠️ 某些表有級聯刪除設定
- 刪除 users → 自動刪除相關 orders, carts
- 刪除 products → 自動刪除相關 images, product_units

### 軟刪除
- 目前 **未使用軟刪除**
- 刪除是 **永久的**
